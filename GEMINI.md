# 선형대수학 정리 프로젝트 지침 (Linear Algebra Project)

이 파일은 본 워크스페이스에서 선형대수학 강의 자료를 정리하고 문서를 생성할 때 준수해야 할 **절대적인 기술 표준**을 정의합니다. 모든 CLI 에이전트는 작업을 시작하기 전 이 문서를 반드시 숙지해야 합니다.

git hub : https://github.com/oscar87657/Linear-algebra.git

## 1. 프로젝트 개요
- **목적**: Gilbert Strang의 *Introduction to Linear Algebra* 교재를 바탕으로 초보자용 선형대수학 튜토리얼 제작.
- **범위**: Chapter 1 (Vectors and Matrices) ~ Chapter 2 (Solving Linear Equations).
- **대상**: 선형대수학을 처음 접하는 완전 초보자 (절차적이고 친절한 설명 지향).

## 2. LaTeX 작성 절대 규칙 (Safety Rules)
수식 렌더링 오류를 방지하기 위해 다음 규칙을 **예외 없이 절대적으로 준수**합니다.

1. **인라인 수식 공백 통제**: `$수식$` 기호와 내용 사이에 공백을 두지 말 것. (O: `$E=mc^{2}$`)
2. **블록 수식 격리**: `$$` 블록은 반드시 위아래로 빈 줄을 둔 독립된 줄에 단독으로 작성할 것, '$'블록은 앞뒤로 한칸씩 뛰어쓰기를 할것.
3. **블록 수식 내 빈 줄 금지**: `$$` 내부에는 어떠한 경우에도 빈 줄(엔터 2번)을 넣지 말 것.
4. **부등호 치환**: `<`, `>` 직접 사용 엄금. 반드시 `\lt`, `\gt`, `\le`, `\ge` 로 치환할 것.
5. **첨자 스코프 명시**: 아래첨자(`_`)와 위첨자(`^`) 뒤의 내용은 항상 중괄호 `{}` 로 묶을 것. (O: `\mathbf{v}_{1}`)
6. **인라인 수식 아래첨자 이스케이프 (중요)**: 마크다운 파서가 아래첨자(`_`)를 이탤릭체로 오인하여 렌더링이 깨지는 오류를 방지하기 위해, **인라인 수식(`$...$`) 내의 언더바 앞에는 반드시 역슬래시(`\`)를 붙여 이스케이프 처리**할 것. (O: `$\mathbf{v}\_{1}$`, X: `$\mathbf{v}_{1}$`)
7. **쉼표 뒤 공백**: 수식 내 콤마(`,`) 뒤에는 반드시 한 칸의 공백을 둘 것. (O: `x, y`)
8. **줄바꿈 기호 보존**: 행렬 및 케이스 구분 시 `\\` 가 증발하지 않도록 주의할 것. (필요시 코드 레벨에서 `\\\\` 사용)
9. **표준 환경 사용**: `amsmath` 표준 환경(`matrix`, `align`, `cases`)만 사용할 것.
10. **목차  LaTeX 사용금지**: 목차부분에 링크가 들어가 있음으로  LaTeX을 사용하면 적용되지 않는다.  다른 방법을 사용하도록한다.
11. **행렬 및 복잡한 수식 블록화**: 행렬(`matrix`, `bmatrix` 등) 이나 복잡한 다중 행 수식은 절대 인라인 수식(`$...$`) 으로 작성하지 말 것. 반드시 독립된 줄의 블록 수식(`$$...$$`) 으로 작성하여 렌더링 오류를 방지할 것.
12. **목차 링크 안정성 (HTML 앵커 사용)**: 제목에 특수문자, 수식, 한국어가 포함된 경우 링크 깨짐을 방지하기 위해, 반드시 제목 바로 위에 `<a id="unique_id"></a>` 형태의 HTML 앵커를 삽입할 것. 목차에서는 `[제목](#unique_id)` 형식으로 영문 고유 ID를 참조하여 링크를 연결할 것. (예: `[1. 개요](#intro)`)

## 3. 한국어 서식 규칙
- **강조 서식 (Bold)**: `**단어**` 뒤에 조사가 바로 붙을 경우 반드시 한 칸 띄울 것. (O: `**벡터** 가`)
- **수식과 조사**: `$수식$` 뒤에 조사가 붙을 경우 반드시 한 칸 띄울 것. (O: `$A$ 를`)
